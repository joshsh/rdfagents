1 Introduction

... RDFAgents adheres closely to Linked Data best practices while replacing the request-response, client-server model of HTTP with the asynchronous and peer-to-peer model of FIPA agents. In analogy to the four Linked Data principles, the basic principles of RDFAgents are as follows:

  1. Use Semantic Web URIs as names for things
  2. Use FIPA identifiers as names for agents
  3. When someone asks an agent about a URI, provide useful information using the standards (RDF, SPARQL)
  4. Include links to other URIs and quotations from other agents, so that they can discover related information.
For example, if a user asks the agent named info@example.org about the thing named http://example.org/cities/Beijing, that agent should respond with an RDF description of Beijing. The description should include links to related things such as China, Beijing universities and historical landmarks, and so on. Furthermore, if the description includes information from multiple sources, such as a news service news@foo.com or a weather service weather@bar.net, this should be indicated through the use of Named Graphs and graph quotation. The user then has the option of contacting those sources directly in order to get more detailed or more up-to-date news or weather information about Beijing.
